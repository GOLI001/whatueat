?
Что такое FSM?
🧠

FSM (Finite State Machine) — машина состояний. Представь, что бот всегда находится в каком-то состоянии: ждёт имя, ждёт возраст, ждёт подтверждение. В зависимости от состояния бот реагирует по-разному на одно и то же сообщение.

Без FSM бот не помнит, на каком шаге он находится с конкретным пользователем. FSM решает эту проблему.

Пример: регистрация в боте
🚀 START нет состояния
→ /start
👤 wait_name ждём имя
→ имя введено
🎂 wait_age ждём возраст
→ возраст введён
✅ DONE сброс состояния
1
Объявляем состояния
🗂

Состояния описываются классом-наследником StatesGroup. Каждое поле — отдельное состояние. Называй их понятно — они отражают, чего бот ждёт от пользователя.

bot.py 
from aiogram.fsm.state import State, StatesGroup

class Registration(StatesGroup):
    wait_name = State()   # ждём имя
    wait_age  = State()   # ждём возраст
💡
Можно создать сколько угодно состояний. Для большого бота делай отдельные группы: Registration, Order, Quiz и т.д.
2
Устанавливаем состояние
🎯

Чтобы перевести пользователя в состояние, добавляем FSMContext в параметры хендлера и вызываем state.set_state().

bot.py 
from aiogram.fsm.context import FSMContext

@dp.message(Command('start'))
async def cmd_start(message: types.Message, state: FSMContext):
    await message.answer('Привет! Как тебя зовут?')
    await state.set_state(Registration.wait_name)  # переходим в первое состояние
3
Обрабатываем шаги диалога
💬

Добавляем Registration.wait_name в декоратор — хендлер сработает только когда пользователь находится в этом состоянии. Данные сохраняем через state.update_data().

bot.py 
# Шаг 1: получаем имя
@dp.message(Registration.wait_name)
async def get_name(message: types.Message, state: FSMContext):
    await state.update_data(name=message.text)  # сохраняем имя
    await message.answer(f'Отлично, {message.text}! Сколько тебе лет?')
    await state.set_state(Registration.wait_age)  # следующий шаг

# Шаг 2: получаем возраст
@dp.message(Registration.wait_age)
async def get_age(message: types.Message, state: FSMContext):
    data = await state.get_data()  # достаём сохранённые данные
    name = data['name']

    await message.answer(
        f'Готово! ✅\n'
        f'Имя: {name}\n'
        f'Возраст: {message.text}'
    )
    await state.clear()  # сбрасываем — диалог завершён

⚠️
state.clear() — всегда вызывай в конце диалога, иначе пользователь «застрянет» в состоянии навсегда.
🔀 Ветвление if / else
4
Разные ответы на разные данные
🔀

Внутри хендлера пишешь обычный Python — if / elif / else. Например, проверяем возраст и даём разный ответ:

if age < 14
🚫 «Тебе нет 14 — регистрация недоступна»
elif age < 18
⚠️ «Ты несовершеннолетний, нужно согласие родителей»
else
✅ «Добро пожаловать, всё ок!»
bot.py — хендлер с проверкой 
@dp.message(Registration.wait_age)
async def get_age(message: types.Message, state: FSMContext):
    # Проверяем что введено число
    if not message.text.isdigit():
        await message.answer('❌ Введи возраст цифрами, например: 17')
        return  # остаёмся в том же состоянии

    age = int(message.text)
    data = await state.get_data()
    name = data['name']

    if age < 14:
        await message.answer('🚫 Тебе нет 14 — регистрация недоступна.')
        await state.clear()
    elif age < 18:
        await message.answer(
            f'⚠️ {name}, тебе {age} лет.\n'
            f'Нужно согласие родителей.'
        )
        await state.clear()
    else:
        await message.answer(
            f'✅ Добро пожаловать, {name}!\n'
            f'Возраст {age} — всё ок.'
        )
        await state.clear()
✅
Трюк с return без изменения состояния — если данные неверные, просим ввести снова. Пользователь остаётся на том же шаге.
5
Команда отмены /cancel
🚪

Всегда давай пользователю возможность выйти из диалога в любой момент. Хендлер без фильтра состояния с StateFilter('*') поймает команду в любом состоянии.

bot.py 
from aiogram.filters import Command, StateFilter

@dp.message(Command('cancel'), StateFilter('*'))
async def cancel_handler(message: types.Message, state: FSMContext):
    current = await state.get_state()
    if current is None:
        await message.answer('Нечего отменять — ты в главном меню.')
        return

    await state.clear()
    await message.answer('❌ Действие отменено. Возвращаю в меню.')
ℹ️
StateFilter('*') означает «любое состояние, включая None». Без него команда /cancel работала бы только вне FSM.
📄 Полный код
★
Всё вместе — рабочий бот с FSM
🤖

Скопируй целиком, вставь свой токен — и запускай:

bot.py — полный файл 
import asyncio
from aiogram import Bot, Dispatcher, types, F
from aiogram.filters import Command, StateFilter
from aiogram.fsm.context import FSMContext
from aiogram.fsm.state import State, StatesGroup
from aiogram.fsm.storage.memory import MemoryStorage

TOKEN = 'ТВОЙ_ТОКЕН_СЮДА'

bot = Bot(token=TOKEN)
dp  = Dispatcher(storage=MemoryStorage())


class Registration(StatesGroup):
    wait_name = State()
    wait_age  = State()


# /cancel — работает в любом состоянии
@dp.message(Command('cancel'), StateFilter('*'))
async def cancel_handler(message: types.Message, state: FSMContext):
    if await state.get_state() is None:
        await message.answer('Ты и так в меню.')
        return
    await state.clear()
    await message.answer('❌ Отменено.')


# /start — запускаем диалог
@dp.message(Command('start'))
async def cmd_start(message: types.Message, state: FSMContext):
    await message.answer('Привет! Как тебя зовут? (или /cancel)')
    await state.set_state(Registration.wait_name)


# Шаг 1 — имя
@dp.message(Registration.wait_name)
async def get_name(message: types.Message, state: FSMContext):
    await state.update_data(name=message.text)
    await message.answer(f'Отлично, {message.text}! Сколько тебе лет?')
    await state.set_state(Registration.wait_age)


# Шаг 2 — возраст с проверкой
@dp.message(Registration.wait_age)
async def get_age(message: types.Message, state: FSMContext):
    if not message.text.isdigit():
        await message.answer('❌ Введи число, например: 17')
        return

    age  = int(message.text)
    data = await state.get_data()
    name = data['name']

    if   age < 14: await message.answer('🚫 Тебе нет 14 лет.')
    elif age < 18: await message.answer(f'⚠️ {name}, нужно согласие.')
    else:          await message.answer(f'✅ Добро пожаловать, {name}!')

    await state.clear()


async def main():
    await dp.start_polling(bot)

if __name__ == '__main__':
    asyncio.run(main())
⚠️
MemoryStorage() хранит состояния в памяти — при перезапуске бота они сбрасываются. Для продакшена используй RedisStorage.
📌 Итог урока
🗂

StatesGroup

Класс с состояниями. Каждое поле — отдельный шаг диалога. Создай один класс на сценарий.

🎯

set_state / clear

set_state() — переводит в состояние. clear() — сбрасывает. Всегда сбрасывай в конце диалога.

💾

update_data / get_data

Сохраняй данные пользователя между шагами через update_data(), читай через get_data().

🔀

if / elif / else

Обычный Python внутри хендлера. return без изменения состояния — попросить ввести снова.

🚪

StateFilter('*')

Фильтр для /cancel — ловит команду в любом состоянии. Всегда добавляй эту возможность.

💡

MemoryStorage

Простое хранилище для разработки. Для боевого бота переходи на Redis.
