⌨️ Часть 1: Reply Keyboard
1
Создаём клавиатуру
🛠

Сначала импортируем нужные классы и собираем клавиатуру. Кнопки группируем по строкам — каждый список внутри keyboard — это одна строка.

bot.py 
from aiogram.types import ReplyKeyboardMarkup, KeyboardButton

# Создаём кнопки
btn_catalog  = KeyboardButton(text='📋 Каталог')
btn_about    = KeyboardButton(text='ℹ️ О нас')
btn_contacts = KeyboardButton(text='📞 Контакты')

# Собираем клавиатуру (keyboard — список строк)
keyboard = ReplyKeyboardMarkup(
    keyboard=[
        [btn_catalog, btn_about],  # первая строка
        [btn_contacts]             # вторая строка
    ],
    resize_keyboard=True  # подгоняет размер под экран
)
💡
resize_keyboard=True — всегда добавляй это. Иначе кнопки будут огромными и займут пол-экрана.
2
Отправляем клавиатуру пользователю
📤

Клавиатура передаётся в параметр reply_markup при отправке сообщения. После отправки она остаётся на экране до следующего обновления.

bot.py 
@dp.message(Command('start'))
async def start_handler(message: types.Message):
    await message.answer(
        'Привет! Выбери действие:',
        reply_markup=keyboard  # передаём клавиатуру
    )
3
Обрабатываем нажатие
👆

При нажатии Reply-кнопки бот получает обычное текстовое сообщение — текст кнопки. Ловим его через F.text:

bot.py 
from aiogram import F

@dp.message(F.text == '📋 Каталог')
async def catalog_handler(message: types.Message):
    await message.answer('Вот наш каталог...')

@dp.message(F.text == 'ℹ️ О нас')
async def about_handler(message: types.Message):
    await message.answer('Мы — крутая команда!')

@dp.message(F.text == '📞 Контакты')
async def contacts_handler(message: types.Message):
    await message.answer('Пиши нам: @support')
✅
Готово! Запусти бота, нажми /start — увидишь кнопки внизу экрана. Нажми любую — бот ответит.
★
Полный код — Reply Keyboard
📄
bot.py — полный файл 
import asyncio
from aiogram import Bot, Dispatcher, types, F
from aiogram.filters import Command
from aiogram.types import ReplyKeyboardMarkup, KeyboardButton

TOKEN = 'ТВОЙ_ТОКЕН_СЮДА'
bot = Bot(token=TOKEN)
dp = Dispatcher()

# Клавиатура
keyboard = ReplyKeyboardMarkup(
    keyboard=[
        [KeyboardButton(text='📋 Каталог'), KeyboardButton(text='ℹ️ О нас')],
        [KeyboardButton(text='📞 Контакты')]
    ],
    resize_keyboard=True
)

@dp.message(Command('start'))
async def start_handler(message: types.Message):
    await message.answer('Привет! Выбери действие:', reply_markup=keyboard)

@dp.message(F.text == '📋 Каталог')
async def catalog_handler(message: types.Message):
    await message.answer('Вот наш каталог...')

@dp.message(F.text == 'ℹ️ О нас')
async def about_handler(message: types.Message):
    await message.answer('Мы — крутая команда!')

@dp.message(F.text == '📞 Контакты')
async def contacts_handler(message: types.Message):
    await message.answer('Пиши нам: @support')

async def main():
    await dp.start_polling(bot)

if __name__ == '__main__':
    asyncio.run(main())
🔲 Часть 2: Inline Keyboard
1
Создаём Inline клавиатуру
🛠

У каждой Inline кнопки есть callback_data — скрытая строка, которая приходит боту при нажатии. Пользователь её не видит.

bot.py 
from aiogram.types import InlineKeyboardMarkup, InlineKeyboardButton

inline_kb = InlineKeyboardMarkup(
    inline_keyboard=[
        [
            InlineKeyboardButton(text='🐱 Кошки', callback_data='cats'),
            InlineKeyboardButton(text='🐶 Собаки', callback_data='dogs'),
        ],
        [
            InlineKeyboardButton(text='❌ Отмена', callback_data='cancel'),
        ]
    ]
)
2
Отправляем и обрабатываем callback
📡

Нажатие Inline кнопки вызывает callback_query — не сообщение! Ловим через @dp.callback_query и фильтруем по F.data. В конце обязательно вызываем answer() — это убирает "часики" на кнопке.

bot.py 
from aiogram.types import CallbackQuery

# Отправляем сообщение с inline кнопками
@dp.message(Command('menu'))
async def menu_handler(message: types.Message):
    await message.answer('Выбери категорию:', reply_markup=inline_kb)

# Обрабатываем нажатия
@dp.callback_query(F.data == 'cats')
async def cats_cb(callback: CallbackQuery):
    await callback.message.answer('🐱 Раздел: Кошки')
    await callback.answer()  # убираем часики!

@dp.callback_query(F.data == 'dogs')
async def dogs_cb(callback: CallbackQuery):
    await callback.message.answer('🐶 Раздел: Собаки')
    await callback.answer()

@dp.callback_query(F.data == 'cancel')
async def cancel_cb(callback: CallbackQuery):
    await callback.message.answer('Отменено.')
    await callback.answer('Действие отменено', show_alert=True)
⚠️
callback.answer() — всегда вызывай в конце! Без него кнопка будет "крутить часики" 30 секунд. Можно передать текст — появится всплывающее уведомление.
🎁
Бонус: редактируем сообщение по кнопке
✏️

Вместо отправки нового сообщения можно изменить текущее — это выглядит профессиональнее. Используем edit_text():

bot.py 
@dp.callback_query(F.data == 'cats')
async def cats_cb(callback: CallbackQuery):
    # Редактируем то же сообщение вместо нового
    await callback.message.edit_text(
        '🐱 Раздел: Кошки\n\nЗдесь будет список кошек...'
    )
    await callback.answer()
💡
Можно и клавиатуру обновить: edit_reply_markup(reply_markup=new_kb) — это позволяет делать многошаговые меню.
📌 Итог

Что ты теперь умеешь

⌨️
Reply Keyboard
Создать постоянное меню внизу экрана. Кнопки отправляют текст — ловишь через F.text == '...'
🔲
Inline Keyboard
Прикрепить кнопки к сообщению. Нажатие — это callback_query, ловишь через F.data == '...'
✏️
edit_text()
Обновить сообщение по кнопке без отправки нового — для красивых многошаговых меню
⚡
callback.answer()
Всегда вызывай в конце обработчика callback — убирает индикатор загрузки с кнопки
